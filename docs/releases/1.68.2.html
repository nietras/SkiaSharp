Version 1.68.2

Stability, API surface, and new platform views · Released · NuGet · GitHub Release

Highlights

A large release that expands the API surface (SKColorF overloads, more SKImage, SKRegion, and SKCanvas members, and writable pixel spans), introduces GTK views for Xamarin.Forms, and resolves a broad set of view disposal, concurrency, and memory issues. Nine community contributors — including @Gillibald, @mscherotter, and @ziriax — landed user-facing improvements across the views and bindings.

New Features

Engine

  • Updated expat to 2.2.9 — Refreshes the bundled expat native dependency. (#1214)
  • Buffer overflow fixes — Cherry-picks upstream fixes for buffer overflow bugs. (#1037)

GPU & Rendering

  • Improved Perlin noise shader — Adds an SKShader binding to create improved Perlin noise. (#1056)

API Surface

  • Writable pixel spans — Exposes a generic, writable Span<T> over pixel data. (#1242)
  • SKColorF overloads — Adds SKColorF overloads with SKColorSpace support. (#1116)
  • More SKImage members — Rounds out the SKImage API. (#1126)
  • Complete SKRegion members — Adds the remaining SKRegion members. (#1125)
  • More SKCanvas members — Adds additional members to SKCanvas. (#1090)
  • Reflection-free object creation — Removes reflection usage from SKObject creation for better performance. ❤️ @Gillibald (#1209)
  • More .NET-friendly method names — Renames some methods to follow .NET conventions. (#1255)
  • Nullable merge filter overloads — Merge filter factory overloads now accept null arguments. ❤️ @ziriax (#1185)

Platform

  • GTK views for Xamarin.Forms — Adds GTK view support for Xamarin.Forms. (#1089)
  • Multi-targeted Windows.Forms views — SkiaSharp.Views.Windows.Forms is now multi-targeted. (#1082)
  • Newer XF.Tizen overload — Adds the expected overload for newer versions of Xamarin.Forms Tizen. (#1117)
  • Richer touch events — Adds device type (touch, mouse, pen) to touch events, plus pressure and eraser handling. ❤️ @mscherotter (#1191, #1212)
  • VS2019 and .NET Core 3.0 — Upgrades the build to Visual Studio 2019 and .NET Core 3.0. (#1030)
  • Improved native build + Nano Server — Improves the native build process and adds Nano Server support. (#1040)

Bug Fixes

  • GC no longer collects live objects — Prevents the GC from collecting this and pixels during native calls. (#1258)
  • Cleared fields on dispose — Nulls out the field when disposing. Fixes #960. (#1256)
  • No allocation on disposal — Removes an unnecessary allocation during disposal. (#1257)
  • Removed [Preserve] attribute — Drops the obsolete [Preserve] attribute. (#1229)
  • System fonts no longer disposed — Ensures system fonts are not accidentally disposed. (#1224)
  • Concurrency fixes — Resolves a number of concurrency issues. (#1200)
  • Fixed test crashes and WGL deadlock — Fixes unit test crashes and a WGL deadlock. (#1237)
  • Android surface refresh — Refreshes the Android view surface when it is recreated. (#1234)
  • Reworked view disposal — Re-works views to avoid incorrect disposal and releases Skia objects when a view is unloaded. (#1180, #1213)
  • UWP SKSwapChainPanel antialiasing — Fixes broken antialiasing with the UWP SKSwapChainPanel. ❤️ @validvoid (#1086)
  • SKGLViewRenderer disposal — Corrects SKGLViewRenderer disposal. ❤️ @zbyszekpy (#1095)
  • WPF mouse wheel on touch — Handles the WPF mouse wheel during touch events. ❤️ @alexandrvslv (#1079)
  • Weak reference cleanup — TryRemoves instances from WeakReferences. ❤️ @daltonks (#1039)
  • macOS SKGLView fixes — Fixes a few issues with the macOS SKGLView. (#1083)
  • Correct image decoding — Fixes incorrect image decoding. (#1055)
  • HarfBuzz string fixes — Fixes issues with HarfBuzz strings. (#1034)
  • Paint encoding fallback — Changes paint encoding instead of throwing. (#1152)
  • Redraw and load fixes — Queues the redraw (fixes #1074), keeps the view in the hierarchy, calls MakeCurrent before drawing, and prevents a double-load issue (fixes #1118). (#1135, #1143, #1141, #1133)

Plus several build, CI, and documentation improvements.
