Version 1.68.1

Feature & infrastructure release · Released · NuGet · GitHub Release

Highlights

A large release that modernizes the type marshalling layer, adds Span<T> support across the data and text APIs, and introduces a WPF backend for Xamarin.Forms. HarfBuzz and HarfBuzzSharp received a series of updates and new font-function bindings from @Gillibald, and the project moved onto a brand-new Azure DevOps pipeline with a custom P/Invoke generator. Community contributions from @alexandrvslv, @Gillibald, @Odirb, @danien and @newky2k shaped this release.

New Features

Engine

  • HarfBuzz updates — Rolls HarfBuzz forward through v2.5.2 and v2.5.3 for improved text shaping. ❤️ @Gillibald (#872, #904)

GPU & Rendering

  • SKPictureShader binding — Exposes picture-based shaders for tiling and repeating recorded drawing. ❤️ @alexandrvslv (#973)

API Surface

  • Span<T> for data, bitmap data and text blobs — Adds span-based overloads to reduce allocations and copies when working with raw buffers. (#865)
  • SKTypeface.IsFixedPitch — Adds a binding to query whether a typeface is monospaced. (#1019)
  • Reworked managed-native types — Overhauls the managed/native type plumbing for safer, more consistent interop. (#900)

Text & Fonts

  • HarfBuzzSharp font functions — Adds font-function bindings to HarfBuzzSharp for finer-grained shaping control. ❤️ @Gillibald (#881)
  • HarfBuzzSharp 2.6.1 — Updates HarfBuzzSharp to track the latest HarfBuzz. ❤️ @Gillibald (#929)
  • int unicode overloads — Adds integer code-point overloads for a more consistent shaping API. ❤️ @Gillibald (#972)
  • Load HarfBuzz face by table data — Allows constructing a face directly from raw table data. (#862)

Platform

  • WPF backend for Xamarin.Forms — Adds a WPF rendering backend to SkiaSharp.Views.Forms. (#917)
  • watchOS arm64_32 and bitcode — Builds the arm64_32 architecture for watchOS and enables bitcode for watchOS and tvOS. (#967, #971)
  • fontconfig-free Linux builds — Adds support for building SkiaSharp without fontconfig. (#821)
  • Linux export hiding — Adds a build version script that hides the exported symbols of bundled dependencies on Linux. ❤️ @Gillibald (#984)
  • Split desktop projects and packages — Separates the desktop projects and packages for cleaner platform targeting. (#899)

Bug Fixes

  • Fixed bad colors — Corrects incorrect color output in rendered content. (#1023)
  • Freed memory when a view is detached — Releases resources when the view is detached, fixing a leak (#1004). (#1005)
  • Fixed an SKRoundRect memory leak — Stops SKRoundRect from leaking native memory. (#995)
  • Improved UWP GPU view stability — Hardens the UWP GPU view against intermittent failures. (#1024)
  • Fixed the iOS canvas implementation — Corrects the iOS canvas behavior. (#879, #1022)
  • Fixed transparent-bitmap artifacts in ToSKPixmap — Resolves display artifacts when converting transparent bitmaps (#849). ❤️ @Odirb (#863)
  • Raster views no longer allocate 0 bytes — Prevents zero-byte allocations in raster views (#759). (#761)
  • Corrected reference-only color spaces — Fixes color spaces that were created as references only. (#922)

Plus a new custom P/Invoke generator, a fresh Azure DevOps build pipeline, and numerous CI and documentation improvements.
