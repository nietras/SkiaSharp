Version 3.119.1

Metal and Apple platform fixes · Released · NuGet · GitHub Release

Highlights

A servicing release focused on Apple platform stability — fixing Metal rendering on iOS 15 and lower, MacCatalyst canvas opacity, and disposal crashes — alongside a zlib update and a new PostScriptName on SKTypeface. Most user-facing fixes came from community contributors including @taublast, @jeremy-visionaid, @jonlipsky, and @spouliot.

New Features

Engine

  • Updated zlib to v1.3.0.1+ — Refreshes the bundled zlib native dependency. (#3058)

Text & Fonts

  • PostScriptName on SKTypeface — Exposes the PostScript name of a typeface. ❤️ @jonlipsky (#3263)

Platform

  • WinUI Interop built as /MT — Builds the WinUI interop library with the static runtime. (#3341)
  • EGL/GLESv2 build targets — Builds EGL as win32 and GLESv2 as WASDK. (#3338)

Bug Fixes

  • Metal on iOS 15 and lower — Hotfixes Metal rendering for iOS 15 and earlier. ❤️ @taublast (#3293)
  • SKAutoCanvasRestore disposal crash — Avoids a crash at Dispose when the canvas was already disposed. ❤️ @taublast (#3291)
  • MacCatalyst canvas opacity — Ensures SKXamlCanvas is not opaque on MacCatalyst. ❤️ @spouliot (#3236)
  • Double SKMetalView initialization — Avoids initializing SKMetalView twice. ❤️ @jeremy-visionaid (#3256)
  • UTC DateTime conversion — Fixes UTC DateTime conversion. ❤️ @jeremy-visionaid (#3333)
