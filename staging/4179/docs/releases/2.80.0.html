Version 2.80.0

Major milestone update with Vulkan, ARM, and a text overhaul · Released · NuGet · GitHub Release

Highlights

This is one of the biggest SkiaSharp releases in years, upgrading to Skia milestone 80 (#986) and broadening platform support dramatically. Vulkan backends land across Win32, Android, and Linux; ARM64 comes to Win32 and ARM/ARM64 to Linux; and a brand-new WebAssembly target is introduced. Text processing is rebuilt around a glyph-focused SKFont type. Community contributors @ziriax and @bender2k14 helped drive this release.

⚠️ Breaking Changes

  • Native libraries no longer embedded — libSkiaSharp is no longer embedded inside the managed assemblies, which may require updating deployment configurations. (#1296)
  • SKFont public API is now glyph-only — Text-related members move under SKFont, with other types consuming it under the hood. (#1299)

New Features

Engine

  • Skia milestone 80 — Upgraded the native engine to a much later Skia (m80) with all upstream improvements. (#986)

GPU & Rendering

  • Vulkan for Win32 (plus APIs for everyone) — Full Vulkan backend support on Win32 along with the shared Vulkan APIs. (#1010)
  • Vulkan on all supported platforms — Vulkan native libraries are now built across the supported platform matrix. (#1287)
  • Improved GPU APIs — Streamlined GRContext and related GPU type creation and usage. (#1294)
  • GL ES 3.x preferred — Explicitly requests GL ES 3.x when available. (#1388)
  • Correct shader usage — Fixes the shaders used during rendering. (#1323)

Text & Fonts

  • SKFont text engine — The public SKFont API is now glyph-focused, providing a cleaner text pipeline. (#1299)
  • SKCanvas.DrawTextOnPath — Implemented drawing text along a path. ❤️ @ziriax (#1198)

API Surface

  • SKPath.ToWinding() — Convert paths to the winding fill type. (#1326)
  • SKPaint.ColorF — Get and set colors using floating-point SKColorF. (#1325)
  • Snapshots with bounds — Add surface snapshots constrained to bounds. (#1357)
  • Reduced deprecated native usage — Internal calls moved off deprecated native functions. (#1356)
  • Enum members restored — Enum members dropped during the m80 update were added back to preserve compatibility. (#1285)

Platform

  • WebAssembly support — A new WASM package, WASM build of libSkiaSharp, and WASM unit tests. (#1389, #1359, #1361)
  • Linux ARM32 & ARM64 — Native libraries built for armhf and aarch64, including Raspberry Pi. (#1382)
  • Win32 ARM64 — SkiaSharp now builds for Win32 ARM64. (#1358)
  • Alpine native libraries — Added musl-based Alpine Linux binaries. (#1339)
  • Per-library native loading — Load specific native libraries on desktop/.NET Framework. (#1342)
  • Tizen Clang update — Updated the Clang version used to build for Tizen. (#1386)

Bug Fixes

  • Crash with no GL context — Fixed a crash that occurred when there was no GL context. (#1328)
  • Managed/native disposal — Correctly disposes the managed/native object relationship again. (#1344)
  • Text blob builder lifetime — Keeps the text blob builder alive to prevent premature collection. (#1365)
  • Font set lifetime — Keeps the font set alive to prevent premature collection. (#1362)
  • Dependency prefixing — No longer adds the SkiaSharp prefix to dependencies. (#1331)
  • Issue #1353 — Resolved the reported regression. (#1368)
